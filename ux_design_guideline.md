# UX Design Principles & Review Skill（v2.0）

## 概要
本ドキュメントは、UX設計を属人化させず、再現可能な判断基準として扱うための指針である。  
「ユーザー要求 → UX原則 → ルール → チェック → レビュー手順」を一貫して定義する。

---

# 0. User Requirement（ユーザー要求定義）

## テンプレート
- ユーザー：誰が使うか  
- 状況：どのような場面で使うか  
- 目的：何を達成したいか  
- 成功条件：どうなれば成功か  

---

# 1. UX Principles（原則）

## 基本
- ユーザーは考えたくない
- ユーザーは最短で目的を達成したい
- ユーザーは不安を嫌う
- ユーザーはミスする
- ユーザーは覚えない

## 認知
- 認識されない機能は存在しない
- 適切なタイミングで認知させる
- 複雑さはUIで吸収する
- 理解は段階的に形成される
- 制約は事前に提示する

## 要求適合
- UXは目的達成で評価する
- すべての機能は要求に紐づく
- 不要なものは排除する

---

# 2. UX Rules

- 1画面1目的
- 主要アクションは1つ
- 入力は最小限
- エラーは原因＋解決方法
- 戻れる導線を用意

---

# 3. UX Checklist（簡易）

- 目的達成できるか？
- 最短で完了できるか？
- 機能に気づけるか？
- 次の行動が明確か？
- 不安がないか？

---

# 4. Cart UX

## 原則
- 最終確認の場
- 一目で理解できる
- 修正が簡単
- 合計が明確
- 次の行動が明確

---

# 5. Input UX

## 原則
- 入力は最小限
- 補助されるべき
- ミスは即時検知
- 再入力を減らす

---

# 6. UX Review Skill

## 手順
1. 要求確認
2. フロー確認
3. チェック適用
4. 問題抽出
5. 改善提案

---

# 7. UX Quick Checklist

- 目的達成できるか
- 迷わないか
- 不安がないか
