
# Task / Current Code → Implementation Plan Skill（AI-Optimized）

## 概要
タスクと現状コードベースを入力として、実装方針を決定するための skill。
AI活用を前提とし、「どこをどう変更するか」を再現性ある形で整理する。

---

## 目的
- タスクを実装方針に変換する
- 現状コードに沿った変更方法を決める
- 変更箇所を特定する
- 再利用と新規追加の判断を行う
- 変更順序・影響範囲・リスクを明確にする

---

## 入力
- タスク
- 現状コード（関連ファイル・構造）
- 関連設計
- 制約条件

---

## 出力（必須）
- 実装対象一覧
- 変更対象ファイル候補
- 実装方針
- 新規追加 / 既存修正の判断
- 再利用候補
- 実装順序
- 影響範囲
- 注意点 / リスク
- 未確定事項

---

## 基本原則
- 現状コードを無視しない
- 最小変更で達成する
- 既存構造を壊さない
- 再利用を優先する
- 変更箇所と理由を明確にする
- 未確定事項は明示する

---

- 新規作成か
- 重複回避

## チェックリスト
- 変更箇所が明確か
- 再利用が検討されているか
- 層ごとの整合があるか
- 順序が明確か
- リスクがあるか

---

## NGパターン
- 新規実装に逃げる
- 変更箇所が曖昧
- 依存関係無視
- 未確定事項を隠す
