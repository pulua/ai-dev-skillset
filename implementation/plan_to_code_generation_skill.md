
# Implementation Plan → Code Generation Skill（AI-Optimized）

## 概要
Implementation Plan を入力として、既存コードベースに適合した実装コード（差分）を生成するための skill。
AI活用を前提とし、「既存構造を壊さず、最小変更で実装する」ことを目的とする。

---

## 目的
- Implementation Plan をコードに変換する
- 既存コードに沿った形で変更を加える
- 差分ベースで安全に実装する
- 不要な変更や過剰な抽象化を防ぐ
- 一貫した命名・構造を維持する

---

## 入力
- Implementation Plan
- 現状コード
- 設計情報
- 制約条件

---

## 出力（必須）
- 変更コード（差分ベース）
- 追加コード
- 修正コード
- 削除コード（必要な場合）
- 各変更の意図

---

## 基本原則
- 差分ベースで生成する
- 既存コードを尊重する
- 不要なリファクタをしない
- 命名規約を守る
- エラー処理を省略しない
- バリデーションを省略しない
- 動くコードを出す

## チェックリスト

- 不要変更がないか
- エラー処理があるか
- バリデーションがあるか
- 実行可能か
