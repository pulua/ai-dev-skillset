
# Change Request → Impact Analysis Skill（AI-Optimized）

## 概要
変更要求を入力として、影響範囲・リスク・対応方針を特定する skill。

## 目的
- 変更の影響範囲を明確化
- リスクの事前検出
- 見積もり精度向上

## 入力
- 変更要求
- 現状設計 / コード

## 出力
- 影響範囲（機能 / API / DB / UI）
- リスク
- 対応方針

## 観点
- どの機能に影響するか
- API互換性
- データ影響
- UI変更有無
- 非同期処理影響

## 本質
変更そのものではなく「どこに波及するか」を特定する
